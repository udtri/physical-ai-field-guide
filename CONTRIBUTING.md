# Contributing to Awesome Physical AI

Thank you for your interest in contributing! This list is community-driven and we welcome high-quality additions.

## How to Add a New Resource

1. **Fork** this repository.
2. **Edit** `README.md` and add your resource to the appropriate category.
3. **Submit** a pull request with a clear description of what you're adding and why.

### Format

Each entry should follow this format:

```markdown
| [Project Name](https://link-to-project) ![GitHub stars](https://img.shields.io/github/stars/owner/repo?style=flat-square) | One-line description of what it does. |
```

- For non-GitHub resources, omit the stars badge.
- Keep descriptions concise — one sentence, no period at the end unless it's a full sentence.
- Place the entry in alphabetical order within its category.

## Quality Criteria

To keep this list truly awesome, submissions must meet these criteria:

### Must Have

- **Relevant** — directly related to Physical AI, Edge Computing, Industrial IoT, or Manufacturing AI.
- **Active** — the project should be actively maintained (commits within the last 12 months, or is a stable/established tool).
- **Documented** — has a README or documentation that explains what it is and how to use it.
- **Useful** — provides clear value to practitioners in the Physical AI space.

### Nice to Have

- Open source with a permissive license.
- Has community adoption (stars, forks, citations).
- Includes examples or tutorials.

### Will Not Be Accepted

- Self-promotional links without genuine community value.
- Abandoned projects with no recent activity and no stable release.
- Duplicate entries — check if the resource is already listed.
- Resources behind paywalls with no free tier or open alternative.

## Suggesting a New Category

If your resource doesn't fit an existing category, you may propose a new one. Include:

- A clear category name with an emoji prefix.
- At least 3 resources that would belong in the category.
- A brief explanation of why it deserves its own section.

## Pull Request Template

When submitting a PR, please include:

```
### What are you adding?
[Name of the resource]

### Category
[Which section does it belong to?]

### Why is this awesome?
[1-2 sentences on why this resource is valuable for the Physical AI community]

### Checklist
- [ ] I have read the [contribution guidelines](CONTRIBUTING.md)
- [ ] The resource is relevant to Physical AI / Edge Computing / IIoT
- [ ] The resource is actively maintained
- [ ] I have placed the entry in the correct alphabetical position
- [ ] I have used the correct format (name, link, badge, description)
```

## Code of Conduct

Be respectful, constructive, and inclusive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/) code of conduct.

## Questions?

Open an issue if you're unsure about whether a resource fits or which category to use. We're happy to help!
